Download & install guide

Zero to a working Crucible OS install with local AI sized to your GPU. It's one image, built from source — here's the whole path.

No prebuilt ISO yet. Crucible OS is at the design-and-scaffolding stage — the build pipeline and installer config are execution-tested where they can be, but no ISO has been built and nothing has run on real hardware yet. This is the intended path; treat it as such and cross-check TODO.md. When releases exist they'll appear on the releases page and this step 1–2 becomes "download the ISO."

Before you start

  • A Debian/Ubuntu Linux machine to build on — live-build uses debootstrap + chroot, so it can't build on macOS or Windows.
  • ~20 GB free disk on the build host and a little patience — the build is not quick.
  • A USB stick ≥ 8 GB (it gets erased) for the target machine.
  • A target PC — recent Nvidia RTX (Turing/20-series or newer) or recent AMD gets the best driver + local-AI story.
1

Get the source & run pre-flight

On the Linux build host, clone the repo, install live-build, and run the read-only pre-flight check so the build doesn't fail an hour in on something checkable in seconds.

$ git clone https://github.com/Mr-Pythoneer/crucible-os
$ cd crucible-os
$ sudo apt-get install live-build
$ ./preflight.sh            # build-host readiness (safe to re-run)
2

Build the ISO

Pick the hardware strain — it only sets the default desktop + base packages; every strain still gets all five modes. Output is crucible-os-<strain>.iso.

$ cd iso
$ ./build.sh workstation   # or: laptop | lowspec | server | handheld | cloud
  # … long build … → crucible-os-workstation.iso

Run this on the build host, not a thin laptop — it needs the disk and time.

3

Write it to a USB stick

This erases the stick. On Linux/macOS, find the device with lsblk (Linux) or diskutil list (macOS), then:

$ sudo dd if=crucible-os-workstation.iso of=/dev/sdX bs=4M status=progress oflag=sync

⚠ Replace /dev/sdX with your actual USB device — the wrong device will overwrite the wrong disk. Prefer a GUI if unsure:

  • Any OS: balenaEtcher (point-and-click, hard to get wrong).
  • Windows: Rufus — write in DD Image mode.
4

Boot from the USB

Reboot the target PC and open the one-time boot menu (usually F12, F2, Del, or Esc — it varies by vendor), then pick the USB stick.

Secure Boot: on Nvidia Blackwell (RTX 50-series) and other recent GPUs you may need to turn Secure Boot off for the first boot. The driver installer (step 7) walks you through MOK signing so you can turn it back on afterwards — see blackwell-readiness.

5

Try it live, or install

You boot into a live session first. To install to disk, launch the installer — the Calamares wizard walks through language → keyboard → partitioning (guided or manual) → user account → install. Reboot and pull the USB when it's done.

6

First login — pick your mode

Crucible OS is one install that switches personality on command. Set the mode for what you're doing:

$ distro-modectl switch gaming    # or: ai | server | creative | normal
7

GPU drivers + local AI

Install the GPU driver, then set up local AI — distro-ai-setup detects your GPU, picks the tier that fits, and preloads exactly those models (nothing your card can't run).

$ sudo ./drivers/install-nvidia.sh   # RTX/Blackwell -open module + Secure Boot MOK
  # AMD instead: sudo ./drivers/install-amd-microcode.sh
$ distro-ai-setup --install          # detect tier → install LM Studio/ComfyUI → preload fitting models

Switching into AI mode the first time also auto-detects your tier. Not sure what you'll get? Preview it in the configurator — pick your GPU and it shows the exact models.

8

Verify it works

$ distro-ai-model list      # your tier's model menu
$ distro-ai-model status    # what's loaded + the :8080 server
$ distro-ai-ask "write a haiku about linux"

That's it — local, private AI on your own hardware, plus a distro that retunes itself per task.
