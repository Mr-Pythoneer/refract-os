Download & install guide

Download one image, flash it to a USB from any OS — Windows, macOS, or Linux, and install. Local AI sized to your GPU. Here's the whole path.

Fresh milestone, honest caveats. The ISO builds in CI, boots to a full GNOME desktop in QEMU, and — as of 2026-07-16 — boots on real hardware: a UEFI-only ThinkPad X1 Carbon Gen 13 (Intel Arc 140V), reaching the login screen. Prebuilt images download from the Releases page (no login) — flash from any OS. Still genuinely early, though: the installer has never completed a real install on a real disk, so don't point it at a drive you care about without a backup. Cross-check TODO.md.

Before you start

  • A PC to install on (x86-64) — recent Nvidia RTX (Turing/20-series or newer) or recent AMD gets the best driver + local-AI story.
  • A USB stick ≥ 8 GB — it gets erased.
  • Any computer to flash the USB from — Windows, macOS, or Linux all work. You don't need Linux to install Refract. (Building from source needs Linux, but that's optional — step 2 — and CI does it for you.)
One honest hardware limit. The image is a hybrid BIOS + UEFI ISO — it boots on modern UEFI-only PCs with no firmware fiddling (verified under OVMF in CI, and on a real UEFI-only ThinkPad X1 Carbon). Leave Secure Boot off for now. The one real limit: an Apple-Silicon Mac (M1–M4) can flash the USB but can't boot an x86 OS — install onto a PC, or use an emulated VM (slow).
1

Download the image

Grab the latest prebuilt ISO from the Releases page — no account, no Linux. Pick the strain that matches your machine (workstation is the default; also laptop, lowspec, server, handheld, cloud). Every strain still has all five modes.

The image is published in <2 GB parts (GitHub's per-file cap) — download every .part* for your strain and rejoin them (each release lists the exact parts + rejoin command):

# Linux / macOS — after downloading the .part* files:
$ cat refract-os-workstation.iso.part* > refract-os-workstation.iso
# Windows (cmd): copy /b refract-os-workstation.iso.part00 + ...part01 refract-os-workstation.iso

Each release also ships a .sha256 of the joined ISO — verify it if you like. Got the image? Skip step 2 and jump to flashing.

No spare PC? Try it in a VM instead — see the UTM guide (macOS). It boots the same image; no USB needed.

2

Optional: build it yourself (contributors — skip if you downloaded)

You don't need to do this. The image on the Releases page is built automatically by GitHub Actions, so nobody needs a local Linux box. But if you want to build from source, it needs a Debian/Ubuntu Linux host (live-build uses debootstrap + chroot, so it can't run on macOS or Windows) with ~20 GB free disk — or just click Run workflow on the build-iso action and let CI do it.

$ git clone https://github.com/Mr-Pythoneer/refract-os
$ cd refract-os && sudo apt-get install live-build
$ ./preflight.sh              # build-host readiness (safe to re-run)
$ cd iso && ./build.sh workstation   # → refract-os-workstation.iso
3

Flash it to the USB — Windows, macOS, or Linux

This erases the stick. Easiest on any OS is a point-and-click tool — pick the ISO, pick the USB, go:

  • Any OS (recommended): balenaEtcher — Windows, macOS, and Linux; hard to get wrong.
  • Windows: Rufus — select the ISO and write in DD Image mode.
  • macOS: Etcher, or the Terminal (see below). Note: an Apple-Silicon Mac can make the stick but can't boot it — install onto a PC.

Comfortable in a terminal (Linux/macOS)? Find the device with lsblk (Linux) or diskutil list (macOS), then:

$ sudo dd if=refract-os-workstation.iso of=/dev/sdX bs=4M status=progress oflag=sync

⚠ Replace /dev/sdX with your actual USB device — the wrong device overwrites the wrong disk.

4

Boot from the USB

Reboot the target PC and open the one-time boot menu (usually F12, F2, Del, or Esc — it varies by vendor), then pick the USB stick.

Modern PC won't see the stick? The image is a hybrid BIOS + UEFI ISO, so a UEFI-only machine boots it natively — don't enable Legacy/CSM. If the stick doesn't appear, turn Secure Boot off first, and prefer the entry the boot menu labels UEFI. If it still doesn't show, the write itself is the usual culprit: re-flash and verify the stick's checksum against the ISO before blaming firmware.

Secure Boot: on Nvidia Blackwell (RTX 50-series) and other recent GPUs you may need to turn Secure Boot off for the first boot. The driver installer (step 7) walks you through MOK signing so you can turn it back on afterwards — see blackwell-readiness.

5

Try it live, or install

You boot into a live session first. To install to disk, launch the installer — the Calamares wizard walks through language → keyboard → partitioning (guided or manual) → user account → install. Reboot and pull the USB when it's done.

6

First login — pick your mode

Refract OS is one install that switches personality on command. Set the mode for what you're doing:

$ distro-modectl switch gaming    # or: ai | server | creative | normal
7

GPU drivers + local AI

Install the GPU driver, then set up local AI — distro-ai-setup detects your GPU, picks the tier that fits, and preloads exactly those models (nothing your card can't run).

$ sudo ./drivers/install-nvidia.sh   # RTX/Blackwell -open module + Secure Boot MOK
  # AMD instead: sudo ./drivers/install-amd-microcode.sh
$ distro-ai-setup --install          # detect tier → install LM Studio/ComfyUI → preload fitting models

Switching into AI mode the first time also auto-detects your tier. Not sure what you'll get? Preview it in the configurator — pick your GPU and it shows the exact models.

8

Verify it works

$ distro-ai-model list      # your tier's model menu
$ distro-ai-model status    # what's loaded + the :8080 server
$ distro-ai-ask "write a haiku about linux"

That's it — local, private AI on your own hardware, plus a distro that retunes itself per task.
